HTML

  • The head element contains information about the webpage.
  • The body element represents the physical content shown to the user.
  • HMTL is short for Hypertext Markup Language.
  • The HTML section communicates with browsers on proper encoding and language expected from the webpage.
  • In most cases the encoding language used will be UTF-8.
  • This refers to base 2 language representations and the byte sizes used for each character.
  • HTML is the framework of a webpage and requires a sturdy foundation to build on. Think of it as building a house.
  • It is made of elements and their nested content also known as a Parent/Child relationship.
  • The elements consist of tags which divide information within the webpage.This makes it easier for developers to see what is going on.
  • The elemetns fall into categories of Meta-Related, Content Division, and Semantic Elements.
  • Elements and tags are denoted by angled brackets
  • Attributes are added after a tag to further demonstrate the intention of the content and or elements section.
  • Attributes are always defined at the beginning of the tag and never at the end.
  • DOCTYPE's are demonstrative command's for the browser explaining the contents of it's accompanying attribute. In this case it is HTML
  • Doctype isn't case sensitive, however the following attribute is.
  • The head section is only viewable from a developer eye's and is not intended for the end user.
  • The body section contains elements an end user will find viewable.
  • Of course this is just the beginning and there's a metric ton and more to learn.

CSS

  • CSS is short for Cascading Style Sheets. It handles what design the user will see when opening the page.
  • CSS handles:
    • visuals
    • colors
    • aesthetics
    • layout
    • organization
  • CSS has 3 ways of styling a page
    • inline CSS
    • internal CSS
    • external CSS
  • inline
    • by using the --style-- attribute we can change a single element in a header (--h1--).
    • >> --h1-- style="color:red;(>)Prework Study Guide --h1--
  • internal
    • by using --style-- elements in the --head-- section of html we can embed an entire style sheet.
    • Rules of the sheet only apply to this page and can be configured as needed for this single page.
  • external
    • the most commonly used style by design developers
    • externall CSS allows a link to a style sheet from a separate folder allowing changes to easily be made with a smaler chance of mistake.
  • margin
    • indicates space around the outside of the element-- Outermost box
    • indicates space around the content inside the element-- Inner box.
  • The CSS Box Model
    • CSS is made up entirely of boxes
    • the same way pages are formatted in Word processors
    • there is more to learn about the box model and how to manipulate them for design purposes

Git

  • When operating command line prompts for git, it must always start with >> git.
  • >>git status: checks which branch we are currently working on.
  • >>git checkout -b --placeholder name--: creates a new branch to work off the main branch.
  • Example >> git checkout -b feature/add-html: creates a new branch to work off before merging commits to the main branch.
  • Much more to add here.
  • Github is an opensource software development hub to store programs and let others view and add to them seamlessly.
  • Github has it's own workflow design making code changes and bug fixes alike easier for larger teams to tackle.
  • git itself is a version control system that allows continual updates to occur within a team environment.
  • A git repo, or repository, is a folder that stores all files and their version histories.
  • A git branch is one that moves off the main branch in order to make changes, commits, PR, etc while not immediately effecting the main branches code.
  • Git markdown (.md) allows writing of plain text coding that Github recognizes and translates.
  • Commiting changes are saved records of changes made within a repo.
  • Pull Requests, or PR, are when a developer asks for changes made in another branch to be merged with the main branch. The new branch can still be viewed, changed, and discussed in this state.
  • Issues track the tasks developers should accomplish. Issues are closed when a User Story and Acceptance Criteria are met.
